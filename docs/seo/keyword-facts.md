### 判断

1. **交換留学クラスタは的として小さすぎる**。合計しても月600程度、TPも極小。上位学生への適合度は最高だが、第一弾の主軸には据えられない。
2. **長期インターンは KD61（`長期インターン 大学生` でも KD52）。DR0 の現状では正面突破できない**。TP 9,100〜10,000 と旨みは大きいので、権威獲得後の第二〜三弾に温存する。
3. **`gpa 計算` が突出**：Vol 2,300 / KD 0 / TP 2,000。この規模で KD0 は明確な市場の穴。隣接する `大学 gpa`（1,200 / KD4）も取れる。
4. `gpa 計算` は**ツール検索意図**であり、記事ではなく計算ツールを作るべきKW。章2.2の「高精度診断」型・章3.3の「上位校からリンクされる資産」に合致する。
5. **交換留学の選考基準はGPA**。①交換留学は捨てず、GPAを入口とした下流（出願要件からの送客先）に再配置する。
6. GPA計算ツールの「大学名」入力欄は UX上自然であり、**③併存方針で必要な上位校比率の計測装置**を最小の摩擦で設置できる。

**結論**：上位層向け第一弾を「GPA（計算ツール＋難関大の履修・GPA設計）」に決定。交換留学は第二弾へ。

## 2026-07-21 / 調査3：`gpa 計算` SERP 実態検証

**目的**：Vol 2,300 × KD0 という好条件が本物か、大手が占有していないかの裏取り。第一弾の意思決定がこの1点に依存するため実施。

その他 上位に混在：早稲田大PDF、青山学院大（留学用GPA）、リクナビ、AOKI、berkeleyhouse（留学に必要なGPA）、YouTube。

### 判断

1. **KD0 は実態と一致。参入余地は本物**。上位を占めるのは「汎用計算サイト」「大学の学内PDF」「弱小ツールサイト」で、**ページ単位の権威（UR）は全て 0〜4**。DR0 のドメインでも、URが積み上がれば十分に食い込める構造。
2. **競合ツールは日本の大学事情に対応していない**。カシオは汎用GPA計算、ulabo・chu-pick も単純計算のみ。日本の大学は**GPA換算方式が大学ごとに異なる**（4段階/5段階、秀優良可の扱い、GP係数）ため、**大学別の算出方式に対応した計算機は空白地帯**。ここが差別化点。
3. **「大学名を選ばせる」UIが機能上必然になる**。上位校比率の計測装置を、後付けの設問ではなくプロダクトの必須機能として自然に埋め込める。③併存方針の計測要件をここで満たせる。
4. 上位に **PDF が3本（長崎大・芦屋大・早稲田大）** 入っている。PDFに負けている SERP は、まともなUXのHTMLページで上位を取りやすい典型。
5. **GPA→留学の導線が検索行動として実在する**。SERPに青山学院大「留学用GPA」・berkeleyhouse「留学に必要なGPA」が入っており、第二弾に交換留学を置く設計の裏付けになる。
6. 一方で SERP には**就活文脈（リクナビ・AOKI）が混在**。ProofLoop は就活系アフィリ除外方針（CLAUDE.md §5）のため、この文脈には寄せずに「履修設計・留学出願要件」側で差別化する。

**結論**：`gpa 計算` を第一弾の主軸KWとして確定。差別化の核は「**日本の大学別GPA換算方式に対応した計算機**」。
