## 🔴 最優先（GPA計算機の公開直後にやる）

`/gpa`（GPA計算機）を 2026-07-22 に本番公開しました。狙っている `gpa 計算` は月間2,300検索・難易度0の穴なので、初動が効きます。

- [ ] **Google Search Console で `/gpa` のインデックス登録をリクエストする**
  - 手順：GSC → URL検査 → `https://proofloop.jp/gpa` を入力 → 「インデックス登録をリクエスト」
  - 理由：sitemap 経由のクロールを待つより速い。順位が付くまで時間がかかる施策なので、公開が早いほど有利。
  - あわせて `https://proofloop.jp/sitemap.xml` を再送信しておくと確実。

- [ ] **GA4 にカスタムディメンションを2つ登録する**
  - 登録するもの（いずれも**イベントスコープ**）：
    - `metric_id` … どの指標で計算されたか（GPA / 基本平均点 / 成績評価係数を識別）
    - `input_mode` … 科目ごと入力 / まとめ入力
  - 手順：GA4 → 管理 → データの表示 → カスタム定義 → カスタムディメンションを作成
  - 理由：**登録しないと、送信はされているのにレポートで参照できません。** 上位校比率の測定という施策の目的そのものが達成できなくなります。
  - ⚠️ 注意：従来の `gpa_band` を `value_band` に置き換えました。**既存の `gpa_band` レポートは継続しません。** 過去データが必要なら、置き換え前後で期間を分けて扱ってください。

- [ ] **本番URLで最終目視する**
  - `https://proofloop.jp/gpa`
  - 確認するのは2点だけで十分：大学一覧に東大2エントリが出るか／実際に計算できるか
  - Claude が確認済みなのは `localhost` のみです。

  - 現在このツールで**唯一、出典に明記がなく推論で出している数値**です。画面上では「推論である」「提出先に確認を」と明示しているので誠実な状態ですが、確定できれば注記を外して断定でき、競合に対する優位がさらに強くなります。

  - ② **成績評価係数で不可・F を分母に含めるか**
    同じファイル内で記載が矛盾しています。白紙シートの式は「総**登録**単位数」で除す（＝不可を含む）、記入例シートの手順説明は「不可やFは単位数に含めません」。現在は切替式にして既定を「含める」にしています。
  - 確認が取れたら Claude に伝えてください。データと注記を更新します。

## 参考：Claude 側で対応済み・対応可能なこと
