## 1. 背景と目的

`/gpa` は「日本の大学ごとに異なるGP換算方式に対応し、出典と確認日を明示する」ことを差別化の核として公開した。現在12大学13エントリを収録している。

本specはこれを2点拡張する。

1. **東京大学への対応**。東大は全学でGPAを算出していないため初回リリースでは対象外としたが、実際には進学選択で使う**基本平均点**と、奨学金・交換留学の選抜で使う**成績評価係数**という2つの指標が存在する。上位学生ターゲット（東京一科＋旧帝＋早慶上智ICU）において東大は中核であり、ここが空白のままなのは施策上の大きな穴である。
2. **まとめ入力モード**。現在は科目を1行ずつ入力する形式のみだが、「優が20単位、良が12単位」という粒度で入力できると、学期をまたいだ累積GPAを出したい学生の入力コストが大幅に下がる。

### 1.1 貫く原則（既存実装から継承）

- **推測で数値を作らない。** 出典に記載のない値は計算せず、計算できない旨を伝える。
- **不確かさを隠さない。** 出典が沈黙している点、出典同士が矛盾している点は画面に明示する。
- 出典URLと確認日を画面に表示する。

## 2. 調査結果（2026-07-21 実施）

東大の2指標は性質が大きく異なる。

| | 基本平均点 | 成績評価係数 |
|---|---|---|
| 式 | Σ(評点 × 単位数 × 重率) / Σ(単位数 × 重率) | Σ(評価ポイント × 単位数) / Σ単位数 |
| 範囲 | **0〜100点**（連続値） | **0〜3**（離散） |
| 科目ごとの入力 | 評点（生の0〜100点） | 評語、または点数を4段階へバケット |
| 対象範囲 | **前期課程（2Sセメスター）まで** | 入学後の全学期（学部＋大学院の通算） |
| 重み | 単位数 × **重率**（1 / 0.1 / 0） | 単位数のみ |
| 用途 | 進学選択 | 奨学金・交換留学の選抜 |

**評価ポイント対応表（成績評価係数）**

| 4段階 | 5段階 | 100点満点 | 評価ポイント |
|---|---|---|---|
| 優(A) | S | 100〜90 | 3 |
| 優(A) | A(B) | 89〜80 | 3 |
| 良(B) | B(C) | 79〜70 | 2 |
| 可(C) | C(D) | 69〜60 | 1 |
| 不可(F) | F | 59以下 | 0 |

5段階方式では上位2段が同じ3ポイントに潰れる点に注意。

**重率（基本平均点）**：科目カテゴリごとの必要単位数の枠内なら1、枠を超えた基礎・展開科目は0.1、対象外は0。科類（文一〜理三）ごとに枠が異なる。

**確度**：両指標の式・範囲・対象範囲は東大公式資料（教養学部前期課程PDF、東大がホストする成績評価係数計算表）から直接確認できており高い。一方、**科類別の重率表は完全には取得できていない**。また後述のとおり、成績評価係数の不可の扱いについて**公式資料内に矛盾がある**。

## 3. 設計上の中心的判断

### 3.1 重率は自動計算せず、ユーザーに選ばせる

科類×科目カテゴリの必要単位数の規則をコードに実装すれば入力は楽になるが、**規則が一部しか判明しておらず、改定も入る**。半分しか分かっていない規則を自動化すれば、進学選択という重い意思決定に使われる数値を誤って出すことになる。これは本施策が一貫して避けてきた「推測で埋める」行為そのものである。

したがって、**重率は科目ごとにユーザーが `1 / 0.1 / 0` から選ぶ**。既定は1。UIに重率の説明と、東大公式の計算例PDFへのリンクを添える。

科類別の重率表が確実に入手できた時点で自動化を再検討する（本specのスコープ外）。

### 3.2 成績評価係数の「不可の扱い」は切替式にする

東大公式の計算表ファイル内で記載が矛盾している。

- 白紙シートの式：分母は「総**登録**単位数」→ 不可も分母に算入される
- 記入例シートの注記：「不可やFは単位数に含めません」→ 不可は計算から除外される

どちらか一方を黙って採用すれば、学生は提出先の期待と異なる数値を提出しうる。**切替式にし、既定は「不可を分母に含める」（白紙シートの式・数値が低く出る側）**とする。画面に「公式資料内で記載が矛盾しているため、提出先（奨学金窓口・国際交流課）の指示をご確認ください」と明記し、出典URLを示す。

### 3.3 まとめ入力は評語方式に限定する

まとめ入力は「評語ごとに合計単位数」を入れる方式であり、素点方式（東北大・東京科学大・大阪大・基本平均点）には乗らない。点数帯ごとのまとめ入力も検討したが、東京科学大は `(評点−55)/10` という連続式で帯が存在せず、素点方式の中でも対応可否が割れて説明が破綻する。帯の代表点を勝手に使う案は「推測で埋める」ため却下。

**`method: "grade"` かつ重率を使わない方式のときだけ**まとめ入力を提供する。

## 4. データモデルの変更

### 4.1 `lib/gpa/types.ts`

```ts
/** 指標の計算方式 */
export type ScaleMethod =
  | "grade"  // 評語 → グレードポイント
  | "score"  // 素点 → グレードポイント
  | "raw";   // 評点をそのまま加重平均する（GP換算をしない）

/** 結果パネルのCTA方針 */
export type CtaPolicy =
  | "gpa-threshold"        // 値が3.0以上なら留学、未満なら履修設計
  | "always-study-abroad"  // 値によらず留学ガイドへ
  | "always-credits";      // 値によらず履修ガイドへ

export type GradeScale = {
  id: string;
  label: string;
  method: ScaleMethod;
  grades?: GradeOption[];                    // method === "grade" のとき必須
  scoreToPoint?: (score: number) => number | null;  // method === "score" のとき必須
  /** この指標の満点。GPAなら4や4.3、成績評価係数なら3、基本平均点なら100 */
  maxValue: number;
  /** 画面に出す指標名。"GPA" / "基本平均点" / "成績評価係数" */
  metricLabel: string;
  /** 値に付ける単位。基本平均点なら "点"。無ければ付けない */
  unitSuffix?: string;
  /** 重率の入力欄を出すか。基本平均点のみ true */
  usesWeight?: boolean;
  /** 「不可を分母から除外する」の切替を出す場合の設定 */
  failExclusionToggle?: {
    /** 除外対象とする評語のラベル */
    failLabels: string[];
    /** チェックボックスの脇に出す説明 */
    note: string;
  };
  /** 結果パネルのCTA方針。省略時は "gpa-threshold" */
  ctaPolicy?: CtaPolicy;
  note?: string;
};

export type Course = {
  id: string;
  name: string;
  credits: number;
  grade?: string;
  score?: number;
  /** 重率。usesWeight の方式でのみ使う。省略時は 1 として扱う */
  weight?: number;
};

/** 計算結果 */
export type MetricResult = {
  /** 小数第2位まで四捨五入した指標値 */
  value: number;
  /** 分母に実際に寄与した科目の素の単位数の合計（重率を掛ける前の値） */
  totalCredits: number;
  /** 分母に実際に寄与した科目数 */
  countedCourses: number;
};
```

**重率0の科目の扱い**：重率0の科目は分子にも分母にも寄与しない。したがって `totalCredits` と `countedCourses` からも**除外する**。「算入科目：3科目／合計8単位」と表示しながら実際は2科目分しか計算していない、という食い違いを作らないため。
```

**改名について**：既存の `GradeScale.maxGpa` を `maxValue` に、`GpaResult` を `MetricResult` に、`.gpa` を `.value` に改名する。基本平均点の 68.92 を `gpa` という名前の変数に格納するのは、実態と異なる名前をコードに残すことであり、本施策が画面上で避けてきたことをコード内で行うに等しい。`lib/gpa/` は小さく、テスト29件で固めてあるため改名は安全に通せる。

### 4.2 計算ロジック（`lib/gpa/calculate.ts`）

`method: "raw"` を追加する。

```
値 = Σ(評点 × 単位数 × 重率) / Σ(単位数 × 重率)
```

- 評点は0〜100の範囲。範囲外は既存の `invalid_score` を返す。
- 重率は `1 / 0.1 / 0` のいずれか。それ以外は新設のエラー `invalid_weight` を返す。
- 重率の合計（Σ 単位数 × 重率）が0なら既存の `zero_credits` を返す。
- `usesWeight` でない方式では `Course.weight` を読まない。**既存の grade / score 方式の計算は一切変更しない。**
- 丸めは既存の `roundTo2` をそのまま使う（公式の計算例も小数第2位：68.92）。

**不可除外**は計算ロジックに入れない。UI側で対象科目を除外した `Course[]` を `calculateGpa` に渡す。計算関数を分岐させないため。

### 4.3 まとめ入力の合成（`lib/gpa/bulk.ts` 新規）

```ts
/**
 * 評語ごとの合計単位数から Course[] を合成する。
 * 単位数が0または未入力の評語は除外する。
 */
export function coursesFromGradeTotals(
  scale: GradeScale,
  totals: Record<string, number>
): Course[];
```

合成した `Course[]` を既存の `calculateGpa` にそのまま渡す。計算側にモードの概念を持ち込まない。

## 5. 東京大学の登録

大阪大学を入学年度で2エントリに分けた前例と同じ形を取る。

| id | name | method | maxValue | 特記 |
|---|---|---|---|---|
| `u-tokyo-basic-average` | 東京大学（基本平均点・進学選択用） | `raw` | 100 | `usesWeight: true`、`unitSuffix: "点"`、`ctaPolicy: "always-credits"` |
| `u-tokyo-grade-coefficient` | 東京大学（成績評価係数・奨学金／交換留学用） | `grade` | 3 | `failExclusionToggle` あり、`ctaPolicy: "always-study-abroad"` |

両エントリとも `sourceUrl` と `verifiedAt` を必須で持つ（既存の整合性テストが強制する）。

**note に必ず書く内容**
- 基本平均点：前期課程（2Sセメスター）までの科目が対象であること。重率は科類・科目カテゴリ・必要単位数の枠内かで決まるため、公式の計算例を参照して選ぶこと。指定平均点（学部ごとの平均点）には対応していないこと。
- 成績評価係数：入学後の全学期が対象（大学院生は学部と通算）であること。合格／不合格の2段階評価科目は計算に含めないこと。5段階評価では上位2段がともに3ポイントになること。不可の扱いについて公式資料内に矛盾があること。

## 6. UI設計

### 6.1 結果パネルの一般化

現在「あなたのGPA」「2.65 / 4.3」と固定表示している箇所を、方式が宣言する `metricLabel` / `maxValue` / `unitSuffix` から組み立てる。

- GPA：「あなたのGPA」「2.65 / 4.3」
- 基本平均点：「あなたの基本平均点」「68.92点 / 100点」
- 成績評価係数：「あなたの成績評価係数」「2.40 / 3.00」

### 6.2 重率の入力

`usesWeight` の方式を選んだときのみ、科目行に「重率」のセレクトを追加する（`1` / `0.1` / `0`、既定 `1`）。方式の説明ブロックに重率の意味と公式計算例PDFへのリンクを表示する。

### 6.3 不可除外トグル

`failExclusionToggle` を持つ方式のときのみ、科目入力の下にチェックボックスを表示する。ラベルは「不可（F）の科目を計算から除外する」、脇に `note` を表示。既定はオフ（＝不可を分母に含める）。

### 6.4 入力モードの切替

`method === "grade"` かつ `usesWeight` でないときのみ、「科目ごとに入力 / 成績ごとにまとめて入力」の切替を表示する。

- 2つのモードは独立した状態を持ち、値を相互変換しない（変換すると科目名や個別の点数が失われる）
- モード切替時は結果とエラーをクリアする（大学切替時と同じ扱い）
- まとめ入力中に非対応の方式へ切り替えたら、自動で科目ごと入力へ戻す

まとめ入力のUIは、その方式の評語ごとに「合計単位数」の入力欄を1つずつ並べるだけとする。

### 6.5 CTAの出し分け

`ctaPolicy` に従う。既定は `"gpa-threshold"`（既存挙動）。

| 方式 | ctaPolicy | 理由 |
|---|---|---|
| 通常のGPA | `gpa-threshold` | 既存挙動を変えない |
| 成績評価係数 | `always-study-abroad` | 指標自体が交換留学の選抜に使われるもの。ただし値をもって「要件を満たす」とは主張しない（出典がない） |
| 基本平均点 | `always-credits` | 進学選択の指標であり留学要件ではない。留学を出すのは文脈違い |

就活系のリンク・アフィリエイトは引き続き一切設置しない（CLAUDE.md §5）。

現在「東京大学はGPAを算出していないため対応大学一覧から外している」と記載している段落と、対応するFAQ項目を書き換える。

改訂後の趣旨：**東大は全学としてGPAを算出していないが、進学選択で使う基本平均点と、奨学金・交換留学で使う成績評価係数という2つの指標があり、本計算機はこの2つに対応している。** あわせて、両指標が対象とする範囲（前期課程まで／全学期）が異なることと、いずれも「GPA」そのものではないことを明記する。

## 8. 計測（GA4）

既存の `gpa_calculate` イベントを拡張する。イベント名は履歴の継続性のため変更しない。

```
event: gpa_calculate
params:
  university_id    : string
  university_tier  : string
  metric_id        : string   // 新規。方式のid（指標をまたいだ集計用）
  value_band       : string   // 変更。満点に対する比率で算出する
  course_count     : number
  input_mode       : string   // 新規。"per-course" | "by-grade"
```

`value_band` は `value / maxValue` の比率を次の5段階で区切る。境界は下限を含み上限を含まない（最上位帯のみ1.0を含む）。

| 比率 | value_band |
|---|---|
| 0.00 以上 0.50 未満 | `"0-50%"` |
| 0.50 以上 0.625 未満 | `"50-62%"` |
| 0.625 以上 0.75 未満 | `"62-75%"` |
| 0.75 以上 0.875 未満 | `"75-87%"` |
| 0.875 以上 1.00 以下 | `"87-100%"` |

区切りは既存の絶対値バンド（4.0満点で 2.0 / 2.5 / 3.0 / 3.5）を比率に直したものであり、GPA方式では従来と同じ分布が得られる。これにより満点の異なる指標をまたいで分布を比較できる。**評点・評語・素点そのものは引き続き送信しない。** 成績データをサーバーに保存しない方針も変更しない。

## 9. テスト方針

`lib/gpa/` の純粋関数に対して追加する。既存29件は維持する。

### 9.1 東大公式の計算例を回帰テストにする

調査で確認した東大公式PDF（`zenkyomu.c.u-tokyo.ac.jp` の計算例）に、6科類分の計算例が**評点・単位数・重率つきの表**で掲載されている。結果は 68.92 / 68.92 / 76.82 / 67.91 / 58.59 / 76.08。

**この表をそのままテストケースとして写し、`calculateGpa` の出力が公式の値と一致することを検証する。** 出典に基づく検証としてこれ以上のものはなく、重率の実装が公式の計算と一致することを機械的に証明できる。

### 9.2 その他

- 成績評価係数：評語→ポイント変換、不可除外トグルON/OFF両方の値
- 重率：0.1 と 0 を含むケース、重率合計0でのゼロ除算回避
- `coursesFromGradeTotals`：合成の正しさ、単位数0・未入力の除外
- マスタ整合性テストの拡張：`method: "raw"` の方式は `maxValue` を持つこと、`usesWeight` の方式は `method: "raw"` であること、`failExclusionToggle` の `failLabels` が実在する評語を指していること
- 改名（`maxGpa`→`maxValue`、`gpa`→`value`）後も既存29件が通ること

## 10. スコープ外（明示的に除外）

- **指定平均点**（学部・学科ごとの重率表を用いる平均点）。調査で規則を取得できていない。
- **科類別の重率自動判定**。規則が不完全なため、ユーザー選択とする（§3.1）。
- 留年・再履修の特例処理。
- 素点方式へのまとめ入力。
- **結果の共有機能・ログイン保存機能**。別specで設計する。
- 既存12大学のデータ変更。

## 12. 次のステップ（本specのスコープ外）

- **結果の共有機能とログイン保存機能**（当初要望③）。本specで確定する `MetricResult` と入力データの形を保存対象とするため、順序としてこちらが先。別specを作成する。
- 指定平均点への対応（重率表が入手できた場合）。
- `基本平均点` `成績評価係数` を Rank Tracker の追跡KWに追加。
