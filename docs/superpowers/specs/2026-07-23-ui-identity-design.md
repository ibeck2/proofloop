# ProofLoop 視覚アイデンティティ設計（第一周：土台＋骨格＋トップ）

**作成日：2026-07-23**
**対象：デザイントークン / ヘッダー / フッター / トップページ**

---

## 1. 目的

ProofLoop のUIが「AIが生成した無個性なUI」に見える状態を脱し、学生・学生団体・企業に対して**自社プロダクトとしての視覚的アイデンティティ**を確立する。

第一周は、全ページに効く土台（トークン）と骨格（ヘッダー・フッター）、そして顔であるトップページに限定する。以降のページはこの基準に沿って順次移行する。

## 2. 現状診断（コードから確認した事実）

「AI感」は感覚ではなく、特定可能な5つの原因から出ている。

| # | 症状 | 箇所 |
| --- | --- | --- |
| 1 | ガイド6枠が `blue-50 / emerald-50 / sky-50 / amber-50 / violet-50 / rose-50` の**パステル虹色**。ブランド色と無関係の6色が最も目立つ位置にある | `app/page.tsx:68-73` |
| 2 | アイコンが**2系統混在**（`material-symbols-outlined` と `lucide-react`）。既製セットをそのまま2種類使っている | `components/AppShell.tsx:99` / `components/Footer.tsx:2` |
| 3 | ヒーローが**抽象スローガンの中央寄せのみ**。さらに `<h2>` で始まり `<h1>` が存在しない（SEO上の実害） | `app/page.tsx:12-15` |
| 4 | 淡色の四角＋アイコン＋太字見出しの**同型カード3枚横並び** | `app/page.tsx:21-51` |
| 5 | フッターが `bg-gray-900`（ブランドの紺ではない汎用グレー）。公式SNSリンク2本が `href="#"` の死にリンク | `components/Footer.tsx:8,56,66` |

土台側の問題：

- `tailwind.config.ts` の `colors` に**同一色への別名が10個以上**（`primary`/`navy`/`navy-custom`/`text-main` はすべて `#002B5C`、グレーは6通りの名前）。どれを使うか決まっていないため一貫性が出ない。
- `borderRadius` が全キー `0px` に潰され、別途 `.no-rounded { border-radius: 0 !important }` が存在する二重上書き。
- `globals.css` の `body` が font-family を直接指定しており、Tailwind の `fontFamily` 定義と二重管理。

## 5. デザイントークン

### 5.1 色（6色。既存の別名は段階的に廃止）

| 名前 | 値 | 役割 |
| --- | --- | --- |
| `ink` | `#002B5C` | 紺。見出し・面・ヘッダー・フッター地 |
| `seal` | `#8B0000` | 深紅。**「印」として1画面1〜2箇所に限定**（新歓受付中などの状態表示） |
| `paper` | `#FFFFFF` | 地 |
| `mist` | `#F2F4F7` | 面の切り替え。青みのある紙色（暖色クリームは避ける） |
| `rule` | `#C9D2DC` | 罫線。現行 `#e5e7eb` は無機質なため紺側へ寄せる |
| `graphite` | `#1F2A36` | 本文 |

**深紅の使用ルール（重要）**：現状は装飾として散っており効いていない。**1画面あたり最大2箇所**、状態を示す用途に限定する。面積を絞ることで強くする。

**廃止するもの**：パステル虹色6色（`blue-50`〜`rose-50`）、`bg-gray-900`、色の別名（`navy` / `navy-custom` / `text-main` / `grey-custom` / `secondary-grey` / `neutral-grey` / `neutral-gray` / `text-sub` / `accent-red` / `secondary` ほか）。

### 5.2 書体（3ロール）

| ロール | 書体 | 用途 |
| --- | --- | --- |
| display | **Shippori Mincho B1**（明朝） | `<h1>`・主要セクション見出しのみ |
| body | **Noto Sans JP**（現状維持） | 本文・UI |
| numeric | **Inter**（`tabular-nums`） | 数値（1,958 / 440）・カテゴリラベル |

日本語の明朝を主役にすることが差別化の中心。AI生成デザインの定番は「欧文の高コントラストなセリフ」であり、そことは別の場所を取る。

⚠️ **性能上のリスクと対策**：日本語ウェブフォントは重い。`next/font/google` で `display: swap`、**使用箇所を `<h1>` と主要見出しに限定**する。実装後に LCP を計測し、悪化が大きい場合はフォールバックスタック `"Hiragino Mincho ProN", "Yu Mincho", serif` へ切り替える（0バイトで日本語環境では十分に美しい）。判断基準は §9 に記載。

`globals.css` の `body` への font-family 直接指定は削除し、Tailwind の `fontFamily` に一本化する。

### 5.3 形・罫線・余白

- **角丸は 0 を「意図」として引き受ける。** `tailwind.config.ts` の**全キー `0px` 指定は維持する**。
  - 理由：コード全体に `rounded-lg` 112箇所・`rounded-full` 41箇所・`rounded-xl` 24箇所が書かれており、**現在はこの上書きによってすべて 0 に潰されている**。上書きを外すと第一周のスコープ外の35ページが一斉に角丸化する。
  - `.no-rounded { border-radius: 0 !important }`（`globals.css:45`）のみ削除する。全キー0の指定があるため機能的に不要であり、二重上書きの解消にあたる。
  - 各ページのクラスを `rounded-none` に統一する作業は、ページ移行時に順次行う（第二周以降）。
- 構成要素は**罫線と面**。`rule` 1px を基本、セクション区切りは `ink` 2px。
- 余白は 4 の倍数。セクション間 `64px`(md以上) / `40px`(モバイル)。

### 5.4 アイコン

**`lucide-react` に一本化**する。`material-symbols-outlined`（Google の既製セットをそのまま使っている状態）を全廃する。第一周ではヘッダー・フッター・トップページの範囲のみ置換し、残りは移行タスクとして残す。

### 5.5 ロゴ

`material-symbols` の `loop` アイコンをブランドマークにしている状態を解消する。**「ProofLoop」を Inter の tight tracking で組んだワードマーク**に置き換える。記号は当面持たない。

### 5.6 モーション

- 動くのは**ページ読み込み時に1回だけ**。ループさせない。
- `prefers-reduced-motion: reduce` では**アニメーションを行わず確定状態を即座に表示**する。
- CSS のみで実装（アニメーションライブラリを追加しない）。

## 6. シグネチャ：ヒーロー「顕在化の実演」

ProofLoop の行動指針は「不可視を可視に」である。ヒーローはそれを**主張ではなく実演**する。

### 6.1 構成

```
  見えていなかった力を、
  　100%に。               ← 明朝・大・アニメーションしない

  ━━━━━━━━━━━━━━━━━━━━━━━━━
  慶應義塾大学   ○○合唱団           ▸
  京都大学      △△研究会           ▸
  上智大学      □□学祭実行委員会     ▸
  一橋大学      ※※国際交流会        ▸
  ━━━━━━━━━━━━━━━━━━━━━━━━━
   ↑ 上から順に確定していく（実在の団体名）

  12大学 1,958団体を掲載
  ┌──────────────────────────┐
  │ 大学・キーワードで探す        │
  └──────────────────────────┘
```

### 6.2 各要素の仕様

**`<h1>`：「見えていなかった力を、100%に。」**
ミッションそのものでありながら、直下の実名が即座に裏を取る構造にする。`display` 書体、`ink`、2行組み。**アニメーションさせない**（最初に読む文が消えている状態を作らない）。`学生団体` という検索語はリード文と名鑑セクションの見出しで拾う。

**団体行（4行）**
実在の承認済み団体を4件。大学が重複しないように選ぶ。各行は `大学名`（`graphite`・小）＋ `団体名`（`ink`・中）＋ 遷移記号。行全体が `/organizations/[id]` へのリンク。罫線 `rule` 1px で区切る。

**アニメーション**：各行が `opacity 0 → 1` ＋ `translateY 6px → 0`。継続時間 320ms、`ease-out`、**80ms ずつ遅延**。全体で 600ms 未満に収める。CSS `animation-delay` のみで実装し、JavaScript による制御を行わない。

**実数表示**：「12大学 1,958団体を掲載」。`numeric` 書体、`tabular-nums`。**DBから取得した実数**であり、増えれば自動で増える（記述が嘘にならない）。

**検索フォーム**：1本。送信で `/search` へ遷移（既存の検索実装を流用）。

### 6.3 実装上の制約（守らないとAI感・SEO劣化に転ぶ）

1. **団体名はサーバー側で描画する。** DOMには最初から存在させ、CSSで不透明度のみ変化させる。クライアントで後から流し込む実装にすると、SEOとLCPの両方を損なう。
2. トップページを **Server Component 化**し、ISR（`revalidate = 3600`）でDBから件数と団体4件を取得する。既存の `UpcomingEvents`（client）は子として残す。
3. 表示する4団体は**掲載が新しい順**に、大学が重複しないよう選ぶ。「積み上がっている」ことがそのまま伝わり、時間とともに内容が変わる。
4. 文字化けした `category` を持つ行は除外する。
5. `logo_url` は使わない（1件しか存在しないため）。**ロゴ枠・プレースホルダーを置かない。**

## 7. トップページ全体の構造

```
1  ヘッダー          ワードマーク／アイコン統一／sticky
2  ヒーロー          §6（h1・団体4行・実数・検索）
3  名鑑プレビュー     大学別ブロック。大学名＋所属団体数を帯で。
                    → 「◯◯大学の440団体をすべて見る」
4  カテゴリの入口     運動系 845 / 文化系 496 / 趣味 365 / 学術 234
────────────────────────────────────────
5  大学生活の入口     GPA計算機・単位・お金・一人暮らし・バイト・留学
                    ↑ 検索流入をここで受け、団体側へ戻す導線を張る
6  直近のイベント     既存 UpcomingEvents を流用
7  学生団体の運営者へ  /for-clubs への1行
8  フッター          紺地／死にリンク処理／ガイド導線を追加
```

2〜4が「学生団体と出会う場」という本来の顔、5が検索流入の受け皿。**5を上段に置くと"何のサイトか"が伝わらなくなる**ため、下段に固定する。

**シグネチャの補強**：名鑑ブロック（3）では、大学名を `ink` の帯に置き、横に所属団体数を `numeric` の大きな数字で添える。スクロール時に大学名の帯が sticky で残り、**名簿をめくる感覚**を作る。

**フッター**：`ink` 地に変更。`href="#"` の死にリンクは、URLが確定するまで**リンクとして出さない**（SNSアカウントが未確定なら項目ごと出さない）。ガイド導線（`/baito`・`/guide/circle`・`/guide/study-abroad`・`/guide/credits`・`/guide/money`・`/guide/living-alone`・`/gpa`）を追加する。これは `docs/owner-todo.md` 🟢 の未着手項目を兼ねる。
