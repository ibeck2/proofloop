## 2. ProofLoopが目指す姿（ビジョン・方向性）

### ミッション
**学生団体の潜在能力を顕在化し、持続可能な成長インフラを創る。**
（より広く：全ての大学生のポテンシャルを引き出す）

### 現在地 → 目指す姿

| 現在 | 目指す姿 |
| --- | --- |
| 管理OSとして事務作業時のみ利用（週1〜2時間） | 学生が毎日訪れる価値循環インフラ |
| 限定的な学生団体ツール | 大学生活全般をサポートするメディアプラットフォーム |
| 新歓・タスク管理中心 | 日常的なエンゲージメント＋企業価値の最大化 |

### 3つの行動指針
- **不可視を可視に**：見えない熱量を社会価値へ
- **流動を蓄積に**：代替わりのリセットを、積み上がる知力へ
- **学生からプロへ**：情熱をビジネス水準の実務能力へ

### 4つの能力モデル（機能設計の背骨）

| 能力 | 説明 | 対応機能の例 |
| --- | --- | --- |
| リソース獲得 | 人材獲得と資金調達 | ATS・新歓DX・協賛マッチング |
| 価値創造 | 業務効率化とプロジェクト管理 | タスク管理・イベント・AI自動化 |
| 組織基盤 | ガバナンスと内部管理 | メンバー権限・透明性管理 |
| 資産継承 | ナレッジ蓄積と文化の継承 | 議事録AI・ナレッジベース |

### 事業の二層構造（重要な戦略前提）
ProofLoopは「**学生個人向けメディア（B2C・集客）**」と「**学生団体／企業向けの収益機能（B2B）**」の二層で考える。
- B2C層（ガイド・診断・記事）＝検索流入とSNS拡散で**認知と日常利用**を獲得する入口。
- B2B層（団体管理OS・協賛マッチング）＝**収益**を生む出口。主要収益柱は「実務型スカウト（協賛マッチング）」。
- まずB2Cメディアで流入を貯め、新歓期などにB2B団体獲得へつなげる「仕込み」の発想で動く。

## 5. 守るべき設計方針・ルール（厳守）

- **就活系アフィリエイトは利益相反のため除外**する（学生の利益と相反するため）。
- **canonical / OGP の絶対URLは必ず `https://proofloop.jp`** にする。旧ドメインを絶対に残さない。
- URLはハードコードせず、可能な限り `SITE_URL` 定数（または `NEXT_PUBLIC_SITE_URL`）を一箇所で定義して参照する。
- スクレイピングはMVPでは行わず、**公式URLリンク集方式**を採用（将来的に移行を検討）。
- **ナビゲーションには新コンテンツページを追加しない**。導線は `/guide` ハブ＋フッターで設計する。
- アフィリエイトはバリューコマース（sid: 3766669）。リンクは慎重に扱い、勝手に張り替えない。
  - バイト系：アルバイトEX
  - インターン系：クラウドワークス
- Supabaseのスキーマ変更・RLS・認証設定の変更は影響範囲が大きいため、必ず計画提示と承認を経てから行う。

## 6. 直近の最優先タスク（今ここを進めている）

### ✅ 完了フェーズ（アーカイブ）
- **"B" ドメイン移行の後始末＋計測基盤**：旧Vercel URL置換 → `SITE_URL`一元化、GA4導入、`sitemap.ts`/`robots.ts`、GSC登録・sitemap送信まで完了。
- **Phase 1 コンテンツページ第一弾**：`/guide/credits`・`/guide/money` を実装・公開・sitemap登録済み。

### 🔴 最優先：SEO調査エンジンの立ち上げ（データ駆動のコンテンツ戦略）
計測の「配線」は済んだので、次は**データを見て刺さるSEOを設計する**フェーズ。
- **Ahrefs ＋ DataforSEO の利用を開始**する。Claude（このCLI／MCP連携）から両者を叩き、キーワード分析を回す。
  - Ahrefs は claude.ai MCP 接続済み（Keywords Explorer / GSC / Rank Tracker 等が使える）。DataforSEO は API連携方式を別途検討。
- **分析の観点**：①市場として強い（検索ボリュームの大きい）ワード、②競合が強いワード／弱い（=狙い目の）ワード、を洗い出す。
- **メインペルソナ＝「一人暮らしの大学生」**（※要確定）に刺さるロングテールへ寄せる。credits/money と地続きの通年フック（家賃・自炊・生活費・履修・奨学金・バイト探し）を軸にする。
- 出口：分析結果を下の「AI記事＋SNS生成ループ」の入力キーワードとして流す。

### 🔴 次：AI記事＋SNS（X / Instagram）自動生成でバズを狙う
- Phase 1 の「AI記事自動生成ループ」を、**記事だけでなく X（Twitter）・Instagram 投稿の生成まで**拡張する。
- 記事本文と、そこから派生させたSNS投稿（画像・キャプション・診断系の拡散素材）を半自動生成 → 投稿 → エンゲージメント分析 → リライト、のPDCA。
- 狙いは**バズによる認知獲得**。診断系（シミュレーター・留学診断）を拡散フックとして最大活用。

### 🔴 次：学生団体の画像収集 → 事後通知メールを「営業＆流入の入口」にする
学生団体をProofLoopへ引き込むための、B2C→B2Bの新導線。
- **画像収集**：学生団体の活動画像を、**合法かつ低コスト（API課金の発生しない方法）**で収集する。
  - ⚠️ **要・権利確認**：無断収集後の事後通知は著作権・肖像権・ロゴ権のリスクがある。実装前に必ず方式を確定する。推奨は「**事前許可 / オプトイン方式**」または「公式が公開している素材・許諾済みソースに限定」。少なくとも収集元・利用範囲・削除依頼窓口を明記できる形にする。
- **通知メール**：ProofLoopに掲載した旨をその団体へメール通知。これを**営業メール兼・掲載許可取りメール**として設計し、団体アカウント登録（B2B流入）への入口にする。Resend基盤を流用。
- KPI：通知メール → 団体の返信/登録 のコンバージョン。

### 🔴 併せて：About Us に「全国学生団体連盟」を掲載
- **全国学生団体連盟**（弊社CEOが学生時代に設立した団体）の名称を About Us / 会社紹介に掲載し、学生団体領域での信頼・原体験の裏付けとして提示する。
- 掲載時は団体名・設立経緯・CEOとの関係を正確に。既存トーン（primary/accent）に合わせる。

## 9. 運用・グロースの仕組み化（今後着手予定）

プロダクト実装と並行して、運用・集客の仕組みも構築する。

- **問い合わせ対応**：独自ドメインメール（info@ / support@）の用意。問い合わせを種別（団体導入／学生／協賛企業／アフィリエイト／不具合）で振り分け。一次対応の自動返信＋FAQ誘導。将来的にClaude APIで種別判定→下書き生成→人間が承認送信。
- **SEO調査基盤**：Ahrefs ＋ DataforSEO をClaude連携で運用。市場の強ワード／競合の弱いワード（狙い目）を継続分析し、ペルソナ「一人暮らし大学生」向けキーワードを記事・SNS生成へ供給。
- **記事グロース**：上記Phase 1のAI記事自動生成ループ（解析→生成→計測→改善）をツールまで含めて精緻化。
- **SNS運用（X / Instagram）**：記事から投稿を半自動生成→投稿→エンゲージメント分析→改善。**バズ創出を明確な目標**に置き、診断系（シミュレーター・留学診断）を拡散素材として活用。
- **学生団体流入（画像→通知メール導線）**：合法・低コストで収集した活動画像の掲載通知メールを、営業兼許可取りとして送り、B2B団体登録につなげる（権利面の方式確定が前提）。
- **集客フック戦略の見直し**：マーケットインの時期が当初想定の新歓期からずれているため、季節非依存の通年フック（単位・履修・奨学金・バイト探し）を重視する。`/guide/credits`・`/guide/money` はこの観点でも優先度が高い。
